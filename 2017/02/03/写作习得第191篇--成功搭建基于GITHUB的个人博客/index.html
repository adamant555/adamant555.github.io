写作习得第191篇--成功搭建基于GITHUB的个人博客

By adamant555

发表于 2017-02-03

文章目录

摘要:
从今天开始，我将在独立博客上同步发布写作写得系列文章，以及读书笔记、投资感悟等；

正文:

写作习得系列到今天已经第191篇，我又有新的想法。

写作系列文章主要发布在微信公众号上，方便朋友阅读，最近同时发布在简书上，但个人对简书的兴趣不大，估计不会坚持很久。

微信公众号传播最广泛，阅读最方便，但有几个问题，一是不适合深度阅读，再精彩的文章，放在手机上阅读，坚持不了5分钟，眼镜就要罢工，无奈只能快速翻到文章结尾，看看结论，因此，手机上容易养成快餐阅读习惯，不利于逻辑思维的培养，提醒自己要避而远之。好在我找到比较好的应对方法，把微信公众号文章，通过亚马逊公众号推送到kindle上仔细阅读，阅读体验接近纸质书，而且结合kindlemate能整理标注，感谢和敬佩亚马逊，它在这个小众市场的深度耕耘，已经形成难以匹敌的竞争力。

第二个问题，微信公众号的文章很难整理，我发了一堆文章，想分分类，发现十分困难，即使能分类，也没人愿意在手机上翻看历史文章。因此，微信公众号，只能作为一个展示窗口，文章的长期收集整理需要换个地方。

类似博文一样的文章，最适合放在博客里，国内的新浪网易腾讯都有类似的博客服务，虽然免费易用，但是弊端很明显。界面花里胡哨、广告满天飞，来访的游客心态浮躁，是在不适合张贴学习、思考一类的文章。

阅读一些牛人文章，我发现有GITHUB这个网站，不仅可以免费托管代码，还可以托管网页，而且能像黑客那样通过几个命令，上传自己的文章到博客上，太酷了。未来的趋势是智能化，而智能化的背后是代码，代码逐渐渗透到我们的生活，我们不能对代码视而不见，要利用一切机会，体验代码带给我们的变化，体验代码能把我们带到什么地方。

我花了一天的时间学习搭建HITHUB+HEXO博客，中间遇到很多问题，我利用搜索引擎，一个个解决掉，终于成功发布“共读社”博客。界面还有很多需要装饰的地方，域名也没有买，不过不影响文章的发布和读者的阅读，最重要的一点，我已经开始在做，在此基础上一定能做得更好！而且，会成为年底甚至10年后值得回顾的一件事。

从今天开始，我将在独立博客上同步发布写作写得系列文章，以及读书笔记、投资感悟等。共读社网址：https://adamant555.github.io/
